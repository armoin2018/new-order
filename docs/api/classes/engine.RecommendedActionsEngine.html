    Class RecommendedActionsEngine

    Engine that evaluates, scores, and ranks candidate actions for the most viable victory paths, producing structured recommendations consumed by the Strategic Pathfinder UI panel.

    All formula weights and thresholds are drawn from the injected RecommendedActionsConfig (a slice of GAME_CONFIG.advisory).

    Usage

    const engine = new RecommendedActionsEngine(GAME_CONFIG.advisory);
    const assessment = engine.generateRecommendations(
      factionId, turn, paths, candidateActions, trajectory,
    );

    computeConsistency

    • computeConsistency(
          recentActions: readonly ConsistencyActionEntry[],
          windowSize?: number,
      ): ConsistencyResult

      Compute the strategy consistency score over a sliding window of recent actions.

      Algorithm:

      1. Take the last windowSize entries from recentActions.
      2. Count the frequency of each alignedVictoryPath.
      3. The dominant path is the most frequent value.
      4. score = (dominantCount / totalActions) × 100. If no actions exist the score defaults to NEUTRAL_CONSISTENCY_SCORE (50).
      5. Derive focus bonus / drift penalty:
        • Focus bonus applies when score / 100 > focusBonus.consistencyThreshold (0.7).
        • Drift penalty applies when score / 100 < driftPenalty.consistencyThreshold (0.3).
      6. Effectiveness modifier: +0.05 (focus), -0.05 (drift), or 0.
      7. Popularity decay: -2 per turn on drift, 0 otherwise.

      Parameters

      • recentActions: readonly ConsistencyActionEntry[]

        Ordered list of actions the player has taken.

      • windowSize: number = ...

        Number of most-recent entries to consider (default from config).

      Returns ConsistencyResult

      A ConsistencyResult with score, dominant path, and modifiers.

    • generateRecommendations(
          factionId: FactionId,
          turn: TurnNumber,
          paths: readonly VictoryPathViability[],
          candidateActions: readonly ActionCandidate[],
          trajectory: Record<string, number> | null,
      ): RecommendationAssessment

      Generate a complete RecommendationAssessment for a faction.

      1. Selects the top 3 non-foreclosed victory paths via selectTopPaths.
      2. Ranks candidate actions for each selected path via rankActionsForPath.
      3. Determines advisory confidence:
        • Low — if trajectory is null (insufficient data).
        • Medium — if fewer than 3 non-foreclosed paths exist.
        • High — otherwise.

      Parameters

      • factionId: FactionId

        The faction receiving recommendations.

      • turn: TurnNumber

        The current turn number.

      • paths: readonly VictoryPathViability[]

        All current victory path viabilities.

      • candidateActions: readonly ActionCandidate[]

        Pool of candidate actions to rank.

      • trajectory: Record<string, number> | null

        Optional trajectory data; null signals insufficient data.

      Returns RecommendationAssessment

      A fully-populated recommendation assessment.

    scoreAction

    • scoreAction(action: ActionCandidate): number

      Compute a composite score for a single ActionCandidate.

      Formula:

      compositeScore = (expectedImpact × 2.0) − (resourceCost × 0.5) − (riskLevel × 1.0)

      The result is clamped to the range [0, 100]. High-impact, low-cost, low-risk actions score highest.

      Parameters

      • action: ActionCandidate

        The action candidate to score.

      Returns number

      A composite score in the range [0, 100].
