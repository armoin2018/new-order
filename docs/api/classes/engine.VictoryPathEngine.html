    Class VictoryPathEngine

    Victory Path Viability Scoring Engine.

    Evaluates every victory condition available to a faction, producing a TurnViabilityAssessment per turn. The assessment includes a quantitative score (0–100), qualitative label, directional trend, estimated turns to victory, and a confidence level.

    The engine is stateless — all prior-turn information must be supplied explicitly as method parameters, keeping the engine deterministic and trivially testable.

    See

    • FR-1401 Victory Path Viability Scoring
    • FR-1413 Composite Strategy Score
    • DR-114 Data requirement: VictoryPathViability / TurnViabilityAssessment

    Example

    const engine = new VictoryPathEngine(GAME_CONFIG.advisory);
    const score  = engine.computeViabilityScore(metrics, reqs, turnsLeft);
    const label  = engine.classifyViability(score);

      Produce a complete TurnViabilityAssessment for a faction on a given turn.

      Iterates every supplied victory condition, computing the viability score, label, trend, estimated turns to victory, and confidence level. The result is a snapshot suitable for persisting in the advisory data store (DR-114).

      Confidence heuristics:

      • If no trajectory data is available → ConfidenceLevel.Low.
      • If fewer than HIGH_CONFIDENCE_TURNS_THRESHOLD turns remain → ConfidenceLevel.High (near-end accuracy).
      • Otherwise → ConfidenceLevel.Medium.

      Parameters

      • factionId: FactionId

        The faction being assessed.

      • turn: TurnNumber

        Current turn number.

      • victoryConditions: readonly VictoryConditionInput[]

        All victory conditions to evaluate.

      • currentMetrics: NationMetrics

        Current nation metric values.

      • previousAssessment: TurnViabilityAssessment | null

        The assessment from the prior turn, or null on the first turn. Used for trend computation.

      • trajectory: NationMetrics | null

        Per-turn metric deltas, or null.

    • computeCompositeStrategyScore(

      ): number

      Compute the composite strategy score from three orthogonal inputs.

      Formula (FR-1413):

      composite = topViability × W_top
                + consistency  × W_con
                + lossMargin   × W_loss

      Weights are sourced from GAME_CONFIG.advisory.compositeStrategy. The result is clamped to [0, 100] and rounded to one decimal place.

      Parameters

      • topViability: number

        Highest viability score across all victory paths (0–100).

      • consistency: number

        Strategic consistency percentage (0–100).

      • lossMargin: number

        Distance from the nearest loss condition (0–100).

      Returns number

      Composite strategy score, rounded to one decimal place.

    computeViabilityScore

    • computeViabilityScore(
          currentMetrics: NationMetrics,
          requirements: VictoryRequirements,
          turnsRemaining: number,
      ): number

      Compute a 0–100 viability score for a single victory condition.

      Algorithm:

      1. For each metric in requirements, compute the fulfilment ratio: clamp(currentValue / targetValue, 0, 1).
      2. Average all fulfilment ratios.
      3. Multiply by a turns bonus factor that scales linearly with remaining turns (capped at 1.0 for ≥ 12 turns).
      4. Scale to 0–100 and round.

      If turnsRemaining is 0 the bonus drops to 0.5 (half credit) rather than zeroing out entirely — recognising that partial progress still has value.

      Parameters

      • currentMetrics: NationMetrics

        Current nation metric values.

      • requirements: VictoryRequirements

        Victory condition metric targets.

      • turnsRemaining: number

        Turns left in the game (≥ 0).

      Returns number

      A viability score in the range [0, 100].

    estimateTurnsToVictory

    • estimateTurnsToVictory(
          currentMetrics: NationMetrics,
          requirements: VictoryRequirements,
          trajectory: NationMetrics | null,
      ): number | null

      Estimate the number of turns required to satisfy all metrics of a victory condition, given a per-turn trajectory.

      Algorithm:

      • For each metric, compute the remaining gap (target − current, floored at 0).
      • If the gap is positive and the trajectory for that metric is positive, estimate turns = ⌈gap / trajectory⌉.
      • If the gap is positive but trajectory is zero or negative, the condition cannot be reached → return null.
      • If no trajectory data is available at all (trajectory is null), return null.
      • If all gaps are already zero (condition met), return 0.
      • Otherwise return the maximum of all individual turn estimates.

      Parameters

      • currentMetrics: NationMetrics

        Current nation metric values.

      • requirements: VictoryRequirements

        Victory condition metric targets.

      • trajectory: NationMetrics | null

        Per-turn metric deltas (progress rate), or null if trajectory data is unavailable.

      Returns number | null

      Estimated turns to victory, or null if unreachable.
